# Monolith vs Microservices

## Monolith
Single deployable backend application.

### Advantages
- simpler development
- easier debugging
- easier deployment initially
- lower operational complexity

### Disadvantages
- scaling entire app together
- tighter coupling
- harder large-team coordination

## Microservices
Application split into independent services.

### Advantages
- independent scaling
- service isolation
- technology flexibility

### Disadvantages
- distributed system complexity
- networking overhead
- deployment complexity

## SemanticFeed Decision

SemanticFeed will initially use a modular monolith architecture because:
- project is early-stage
- development speed matters
- operational simplicity is important
